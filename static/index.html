01 What We Built

Crisis Room is an OpenEnv-compliant RL environment where an LLM agent learns to manage cascading production system failures — investigating root causes, issuing remediation commands, and communicating to stakeholders — all within a step budget that simulates real time pressure.

The environment exposes 3 difficulty levels, 8 action types, 4 independent reward components, and a full anti-reward-hacking suite. Runs as a FastAPI server, validates cleanly with openenv validate, deployed live on HF Spaces.

03 Reward Function

Four independent components. No single signal can be gamed without the others pulling it down. 80% programmatic — reward-hack resistant.

Resolution
Did the agent fix the right service with the right action? Verified against hidden ground truth. services_restored ÷ services_affected.
40%
Investigation
Did the agent check relevant logs and run relevant diagnostics before acting? Measured against actual log keys. Rewards thoroughness, not lucky guesses.
30%
Efficiency
Fewer steps = higher score. Resolving in 4 steps earns full marks. Burning all 12 earns near zero. Trains urgency without sacrificing accuracy.
20%
Communication
Was the team notified? Was the incident escalated on hard difficulty? Trains the communication habits that distinguish good commanders from great ones.
10%
−0.05 Wrong escalation — paging on-call without correct diagnosis first
−0.10 Wrong restart, wrong rollback, or redundant notification
SPAM GUARD 3 identical consecutive actions → automatic penalty. No free loops.
TRUNCATED action.target truncated to 500 chars — oversized payloads blocked
REWARD FORMULA
reward = (0.40 × resolution_score) # right service, right fix — vs hidden ground truth
       + (0.30 × investigation_score) # log keys + diagnostic names checked
       + (0.20 × efficiency_score) # fewer steps = higher score
       + (0.10 × communication_score) # notify_team + escalate on hard
       − (0.10 × wrong_restarts) # wrong fix / redundant action / unknown type
04 API Endpoints
POST /reset Load a new incident scenario. Returns initial observation.
POST /step Send an action. Returns observation, reward, done flag.
GET /tasks List available incident scenarios by difficulty.
GET /health Environment health check — returns status + version.
# Start a hard incident
curl -X POST https://hemankkk-crisis_room.hf.space/reset \
  -H "Content-Type: application/json" \
  -d '{"difficulty": "hard"}'

# Run a diagnostic
curl -X POST https://hemankkk-crisis_room.hf.space/step \
  -d '{"action": {"action_type": "run_diagnostic", "target": "dns_check"}}'

# Resolve the incident
curl -X POST https://hemankkk-crisis_room.hf.space/step \
  -d '{"action": {"action_type": "restart_service", "target": "dns-resolver"}}'
